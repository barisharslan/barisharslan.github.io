Tapatat

I actually discovered Patatap last year, and playing around with it helped motivate me to start writing my own music using my computer. So it was a pleasant surprise seeing that there was a project on my Udemy Web Development Bootcamp class that was to make a clone of this awesome little site.

I started off following the instructor's format, but ended up going all out and making a unique animation for every sound. It's not completely done yet, and the code is very messy right now. Once all animations are done, my plan is going to be to go back and refactor everything, following what I would say a class based approach to generate the animations.

I used paper.js to generate the animations. It took a bit to understand the flow of the library, which did end up leading to a not so efficient first attempt. Now that I've really taken in how paper.js works, I am confident in improving this project in efficiency and quality. I also used howler.js to handle the sounds being played, which is so simple I didn't run into a single problem.

The Great RGB Guessing Game

This was also a project for my Udemy class, which I tried my best to improve and make unique. This is actually my second attempt at the project. When I saw the project, I decided to give it a shot without following along with the instructor, which ultimately led to what you would call "spaghetti code". It got so complicated I forgot that all I was making was a simple guessing game.

So I gave up and started over, following the instructor. I caught many problems I had overlooked the first time and came out understanding Javascript a great deal more. I finally gave it another shot using jQuery, and also added a difficulty selector, and it gets tough! All that is happening is that the range of possible colors for each square gets tighter and tighter as the difficulty goes up. At the insane difficulty, there is only a range of ± 10 for each value.
